# 模块：map（通勤测算）

## 负责什么

提供两种通勤数据：
1. **精算**：调用高德/百度地图 API，获取真实路径规划时间（准确但消耗 API 配额）
2. **估算**：优先查历史精算均值，无数据时用离线公式（haversine 直线距离 × 迂回系数），零 API 调用

## 包含文件

| 文件 | 作用 |
|------|------|
| `engine.py` | 精算调度器：高德优先 → 百度兜底，结果写入 commute_store |
| `offline_estimator.py` | 估算：先查 commute_store baseline，无数据再用离线公式 |
| `geo_filter.py` | 地理过滤：对所有房源做 geocode + 直线距离粗筛 + 离线估算 |
| `amap.py` | 高德地图 API 客户端（geocode、transit、riding、walking） |
| `baidu.py` | 百度地图 API 客户端（同上） |
| `address_cleaner.py` | 地址清洗工具：提取候选地址、haversine 距离计算 |

## 数据优先级（估算时）

```
① commute_store.get_stable_baseline()  已固化稳定路线 → 直接用，不调API
② commute_store.get_baseline()         历史精算均值   → 比离线公式准
③ 离线公式（haversine × 迂回系数）     无历史数据兜底 → 误差 ±15~35%
```

## 数据优先级（精算时）

```
① 高德 geocode + 路径规划（默认）
② 百度 geocode + 路径规划（高德失败兜底）
③ 返回失败原因（配额耗尽 / 地址解析失败）
```

## 固化机制（is_stable）

当一条路线满足以下条件时，标记为「已固化」，精算时直接跳过 API：
- 步行 / 骑行：历史样本 ≥ 3 次，且变异系数 CV < 10%
- 公交 / 地铁：历史样本 ≥ 5 次，且变异系数 CV < 20%

固化后，该路线估算值等同于精算值，不再消耗 API 配额。

## 关于数据准确性

高德/百度返回的是「当前时刻」的路线规划预估值，不是用户实际乘车时长。
- 早晚高峰 vs 非高峰差异可达 ±30~50%
- 7日均值是「不同时刻多次测量的平均」，对通勤时长有参考价值，但建议前端标注采集时间
- 骑行/步行数据较稳定；公交/驾车数据受时段影响较大
