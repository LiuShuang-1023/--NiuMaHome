# 模块：ranker（排序与评分）

## 负责什么

对通过初步筛选的房源进行综合评分和排序，同时做最终的硬过滤（车位/价格超限）。

## 包含文件

| 文件 | 作用 |
|------|------|
| `engine.py` | 核心排序引擎：过滤 + 评分 + 排序 + 生成 `Recommendation` |

## 工作流程

### 1. 硬过滤（直接踢出，不展示）

| 过滤条件 | 逻辑 |
|----------|------|
| 车位/储藏室 | 标题含车位关键词，或面积≤16㎡且价格≤700 |
| 价格超上限 | 全包成本超过用户设定的总成本上限 |
| 价格低于下限 | 基础租金低于用户设定的最低价（避免虚假房源） |
| 硬性排除词 | 用户明确说「不要xxx」的关键词 |
| 合租排除 | 用户选「整租」时排掉所有「合租」房源 |

### 2. 软过滤（不踢出，降权到末位）

- 精算通勤超过用户限制：给 `-1.0` 分，排到最后，并标注「⚠️ 精算通勤超出限制 X 分钟（仍展示供参考）」
- 只有精算数据才触发软过滤；离线估算误差大，超限仍硬过滤

### 3. 评分（0~1分，越高越好）

```
综合模式：成本×0.3 + 通勤×0.3 + 置信度×0.2 + 偏好匹配×0.2
价格优先：成本×0.7 + 通勤×0.1 + 置信度×0.1 + 偏好×0.1
通勤优先：通勤×0.7 + 成本×0.1 + 置信度×0.1 + 偏好×0.1
面积优先：面积×0.6 + 成本×0.15 + 通勤×0.15 + 置信度×0.1
```

- 无通勤数据的房源通勤分为 0（必然沉底）
- 所有分量均做归一化（最小-最大标准化）
