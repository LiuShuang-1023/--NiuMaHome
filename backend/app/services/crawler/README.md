# 模块：crawler（房源爬取）

## 负责什么

从链家、贝壳两个平台抓取租房列表页数据，返回标准化的 `Listing` 对象供后续处理。

## 包含文件

| 文件 | 作用 |
|------|------|
| `lianjia.py` | 链家租房爬虫，解析列表页 HTML，提取房源信息 |
| `beike.py` | 贝壳租房爬虫，逻辑与链家相似但字段结构略有差异 |
| `region.py` | 城市/区域名称工具，负责汉字转拼音（生成 URL 路径用） |

## 工作流程

1. 接收用户需求（城市、区域、价格范围）
2. 构造对应平台的搜索 URL
3. 发送 HTTP 请求（携带 Cookie，模拟已登录用户）
4. 解析 HTML/JSON，提取：标题、价格、面积、户型、小区名、图片URL、详情链接等
5. 返回 `list[Listing]` 对象

## 与缓存的关系

- 爬取前先查 `storage/daily_cache.py` 的 `cache_listings` 表
- 命中则直接返回缓存，不发网络请求
- 新抓到的数据写入 `cache_listings`，当天有效
- 次日缓存自动清理，重新抓取最新数据

## 注意事项

- 链家/贝壳对爬虫有反制，Cookie 过期需手动更新（在 `.env.local` 里配置）
- 需安装 `brotli` 包，否则无法解压响应体（`pip install brotli`）
- 请求间隔建议 0.3~1s，避免被封
