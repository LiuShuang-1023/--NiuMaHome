## 一、产品定位

### 1.1 一句话定义
**面向打工人的 AI 租房助理**：用一句自然语言描述需求，AI 自动从多平台聚合房源、计算真实成本、测算多维通勤、推荐最优解。

### 1.2 核心价值
- **省时**：一次描述需求，AI 替代手动多平台搜索
- **省钱**：把"标价房租"还原为"真实月支出"，避免隐藏成本
- **精准**：双地图交叉验证通勤，避免被虚假"15 分钟到地铁"忽悠
- **去伪**：AI 识别引流帖、重复房源、信息缺失，主动追问补全

### 1.3 目标用户
- **核心**：大厂员工（有房补、对通勤敏感、决策周期短）
- **扩展**：所有需要在新城市/新区域快速找房的打工人

### 1.4 产品演进路线
| 阶段 | 形态 | 用户 | 商业模式 |
|---|---|---|---|
| Stage 1 (MVP) | Web (自部署) | 作者本人 | 自用 |
| Stage 2 | Web + 桌面客户端 (Tauri) | 开源用户自部署 | 开源免费 |
| Stage 3 | SaaS + 移动 APP | C 端用户 | 订阅制 |

## 二、核心功能

### 2.1 AI 对话式需求理解（产品入口）

**输入**：用户在对话框中用自然语言描述需求
**输出**：结构化需求 JSON + 反问澄清

#### 输入示例
> "我想找通勤广州番禺南村万博 20 分钟以内的整租一居室或者单间，不要城中村不要合租，纯房租 2000 以内，实际全部房租包含其他费用 2700 以内的房子"

#### 解析结构
```json
{
  "destination": {
    "city": "广州",
    "district": "番禺",
    "landmark": "南村万博",
    "address": "广州市番禺区万博中心",
    "geo": {"lng": null, "lat": null}
  },
  "commute": {
    "max_minutes": 20,
    "modes": ["transit", "riding", "walking"],
    "maps": ["amap", "baidu"]
  },
  "rental_type": {
    "include": ["entire_1b1b", "single_room"],
    "exclude": ["shared", "urban_village"]
  },
  "price": {
    "base_rent_max": 2000,
    "total_cost_max": 2700
  },
  "soft_preferences": [],
  "hard_excludes": ["城中村", "合租"]
}
```

#### 反问机制
当 AI 检测到歧义/缺失关键字段时，主动反问 1-3 个问题：
- 通勤模式优先级（公交/骑行/步行）
- "单间"是否包含合租公寓的独立单间
- 全包预算的水电燃气预估值
- 楼层、电梯、有窗、家具家电硬性要求

4. **非结构化平台**（小红书/闲鱼）：关键词搜索 + LLM 抽取 + 仅展示链接

### 2.3 真实月支出测算

**目标**：把 "房租 2000" 还原为 "实际每月 2700"

#### 计算项
| 项目 | 来源 | 默认估算（缺失时） |
|---|---|---|
| 基础房租 | 平台抓取 | - |
| 物业费 | 平台/AI 询问 | 按面积 2 元/㎡ |
| 水费 | 用户自定义 | 30 元/人 |
| 电费 | 用户自定义 | 80 元/人（夏冬 150） |
| 燃气费 | 用户自定义 | 20 元/人 |
| 网络费 | 用户自定义 | 60 元（合租均摊） |
| 中介费 | 平台/AI 询问 | 月租 50% / 12 摊销 |
| 押金资金占用 | 计算 | 押金 × 年化 3% / 12 |
| 服务费/管理费 | 平台 | 0 |

#### 输出
```
基础房租:     ¥2000
物业费:       ¥80   (40㎡)
水电燃气:     ¥130  (单人估算)
网络费:       ¥60
中介摊销:     ¥83
押金占用:     ¥10
─────────────────
真实月支出:   ¥2363
```

### 2.4 多维通勤测算

#### 核心特性
- **双向计算**：家 → 公司 + 公司 → 家（早晚高峰差异）
- **双地图交叉**：高德 + 百度并列显示
- **多模式并列**：公共交通 / 骑行 / 步行 / 步骑混合 / 步地混合
- **细分指标**：
  - 距最近地铁站距离 + 步行时间
  - 公共交通总时长 + 换乘次数
  - 纯骑行时长
  - 纯步行时长

#### 输出示例
```
家 → 公司 (工作日早 8:30 出发)
┌─────────────┬──────────┬──────────┬─────────┐
│  方式       │ 高德     │ 百度     │ 均值    │
├─────────────┼──────────┼──────────┼─────────┤
│ 公共交通    │ 22 min   │ 25 min   │ 23.5min │
│ 纯骑行      │ 18 min   │ 19 min   │ 18.5min │
│ 纯步行      │ 65 min   │ 68 min   │ 66.5min │
│ 步行+地铁   │ 24 min   │ 26 min   │ 25 min  │
└─────────────┴──────────┴──────────┴─────────┘

最近地铁: 万博站 (步行 8 min, 600m)
```

### 2.5 推荐排序引擎

#### 排序权重（用户可调）
- 真实月支出（越低越好）权重 30%
- 通勤时长（越短越好）权重 30%
- 房源可信度（评分高优先）权重 20%
- 偏好匹配度（户型/楼层/采光）权重 20%

#### 硬过滤
- 超出预算 → 直接剔除
- 超出最大通勤 → 直接剔除
- 命中硬性排除标签 → 直接剔除

#### 推荐展示
- Top 10 / Top 20 卡片流
- 每张卡片：图片 + 真实月支出 + 三维通勤 + 可信度评分
- 一键标记：感兴趣 / 排除 / 已联系

### 2.6 AI Agent 自动询问（P1）

**触发条件**：
- 物业费缺失
- 朝向/采光描述模糊
- 楼层/电梯信息缺失
- 价格异常（疑似引流）

**询问通道**：
- MVP：暂不实现，在 UI 上标记"信息不全"
- P1：链家/贝壳的站内 IM
- P2：微信、电话等

**话术模板**：
```
你好，我对 [小区名] [户型] 这套房子很感兴趣，想问下：
1. 物业费是多少？
2. 朝向和采光怎么样？
3. 这个价格是否包含其他费用？
方便的话希望今天能去看房。
```

---

### 2.7 房源去重

- **平台间去重**：同一套房在 5 个平台发布 → 合并为一条
- **去重策略**：地址 + 户型 + 价格区间 + 图片 hash 的组合相似度
- **保留策略**：保留所有平台链接，价格取最低，信息取最全

### 3.2 技术栈

| 层 | 选型 | 理由 |
|---|---|---|
| 前端 | Next.js 14 + TypeScript + Tailwind + shadcn/ui | 现代、可平滑转 RN |
| 后端 | Python 3.11 + FastAPI + Pydantic | LLM 生态最好 |
| LLM | Anthropic Claude (Sonnet 4) | 对话理解强 |
| 数据库 | SQLite → PostgreSQL | MVP 零配置 |
| 缓存 | 内存 → Redis | MVP 零配置 |
| 爬虫 | httpx + BeautifulSoup + Playwright (按需) | 轻量为主 |
| 地图 | 高德 Web 服务 API + 百度 Web 服务 API | 免费额度够 |
| 桌面端 | Tauri (Stage 2) | 比 Electron 轻 |
| 部署 | Docker Compose | 自部署友好 |

## 五、MVP 验收标准

### 5.1 功能闭环
1. ✅ 用户在 Web 对话框输入自然语言需求
2. ✅ AI 解析为结构化 JSON 并反问澄清
3. ✅ 后端从链家公开数据抓取候选房源
4. ✅ 高德 + 百度地图测算双向通勤
5. ✅ 真实月支出计算
6. ✅ 推荐排序后展示 Top 10
7. ✅ 用户可点开详情、标记喜欢/排除

### 5.2 测试用例
**输入**：
> "我想找通勤广州番禺南村万博 20 分钟以内的整租一居室或者单间，不要城中村不要合租，纯房租 2000 以内，实际全部房租包含其他费用 2700 以内的房子"

**预期输出**：
- 解析正确：destination=南村万博、max_commute=20min、price=2000/2700
- 抓取链家番禺区候选 ≥ 20 套
- 通勤过滤后 ≥ 5 套
- 价格过滤后 ≥ 3 套
- 展示卡片包含真实成本 + 双地图通勤

## 六、风险与应对

| 风险 | 等级 | 应对 |
|---|---|---|
| 平台反爬 | 高 | Cookie 注入 + 频控 + 失败降级 |
| 法律合规 | 高 | 仅自用、不商用、Stage 3 前法律审查 |

| LLM 成本 | 中 | 缓存解析结果 + 按需调用 |
| Cookie 失效 | 中 | 检测 + 提醒用户更新 |
